DevOps BTD Pattern for Continuous Deployment

18/Mar 2015 4 min. read

Code Friction and Service Design I created the DevOps Automation Diagram to map out the flow of code from developer to production in a wholly automated manner: better known as continuous deployment. This map represents the software development life cycle, service deployment, and feedback loops that are integral to service design. I did this so that I could figure out where best to start work for my infrastructure as code project and I found a few optimizations by re-factoring the code flow and moving processes forward in the chain of events.
